# Backend-Only AI News Enhancement & English Fix

Improve the quality of news generated by AI by ensuring narrative integration of entities, enforcing a minimum character count, and fixing the missing English translations. This will be achieved entirely within Cloud Functions.

## User Review Required

> [!IMPORTANT]
> All changes are restricted to the backend. The Android app will receive the improved data automatically via Firestore without any updates needed to the APK.

## Proposed Changes

### 1. Backend: AI System Instructions (`functions/src/categories.ts`)
- **[MODIFY]** [categories.ts](file:///C:/AlfaKotlin/functions/src/categories.ts)
    - Update `getCategorySystemInstruction` to include strict narrative rules.
    - **Narrative Integration**: Mandate that "Who, Where, and What" (people, locations, organizations) must be naturally integrated into the Telugu and English paragraphs.
    - **Length Constraint**: Enforce a minimum of 300 characters for Telugu content (Range: 300-380).
    - **English Quality**: Emphasize that English content is mandatory and must be a professional journalistic translation.

### 2. Backend: News Processing Logic (`functions/src/news_handler.ts`)
- **[MODIFY]** [news_handler.ts](file:///C:/AlfaKotlin/functions/src/news_handler.ts)
    - Update `performAIProcessing` validation.
    - **Remove Fallback**: Strictly prevent the code from using Telugu content as a fallback for English fields.
    - **Retry Logic**: If AI fails to provide valid English content or meet the character count, throw an error to allow for retry (handled by trigger).

### 3. Backend: Social & Citizen Processing (`functions/src/geminiService.ts`)
- **[MODIFY]** [geminiService.ts](file:///C:/AlfaKotlin/functions/src/geminiService.ts)
    - Align prompts in `processSocialPostWithAI`, `processCitizenContentWithAI`, and `processContentWithAI` with the new narrative and length rules.
